## ⚠️ 필수 규칙 (AI 작업 시 반드시 확인)

### 1. 수주율 계산은 반드시 `core_calc.py`를 사용할 것
- 새 스크립트에서 필터/지분계산 코드를 **직접 작성하지 말 것**
- `from core_calc import ...` 해서 기존 로직을 가져다 쓸 것
- 필터 수정이 필요하면 `core_calc.py`를 수정하여 전체 반영
- **임시 조회/분석 스크립트에서도 반드시 core_calc 사용** (부산 기관 필터, 현장 필터 누락 방지)

### 2. 현재 적용된 계산 규칙 (core_calc.py) — 11개
| # | 규칙 | 설명 |
|---|------|------|
| 1 | 지분율 파싱 | corpList에서 사업자번호+지분율 추출, N빵 균등분할, 100% 초과 정규화 |
| 2 | dminsttCd 우선 매칭 | **수요기관(dminsttCd)** 기준 매칭 (계약기관 cntrctInsttCd 아님). 매칭 실패 시 dminsttList fallback |
| 3 | 공사현장 소재지 필터 | bid_notices_raw의 cnstrtsiteRgnNm으로 부산 외 현장 배제 |
| 3b | **용역현장 소재지 필터** | **정부/국가공공기관만 적용. 부산시 소속기관은 현장=부산 확정(스킵). 조달요청 API 현장+키워드 필터** |
| 3c | **쇼핑몰 공사자재 현장 필터** | **공사자재(Y)에 대해 기관별 공고 현장 분석 + dlvrReqNm↔bidNtceNm 텍스트 매칭** |
| 4 | 전화번호 지역번호 필터 | 051/070/010이 아니면 타지역 의심 |
| 5 | 계약명 키워드 필터 | 광역시/도 + **시/군 단위 + 서울구 등 104개** 타지역 지명 포함 시 배제 |
| 6 | 낙찰정보 브릿지 | 키워드/전화번호 필터에 걸려도 부산 지역제한 확정이면 복원 |
| 7 | 입찰공고 rgnLmtInfo | JSON에서 부산 지역제한 여부 추가 확인하여 복원 |
| 8 | 비교단위(compare_unit) | agency_master의 compare_unit으로 기관 통합 집계 |
| 9 | 교육청 통합 | 대시보드에서 각급학교+교육행정기관 → "부산교육청" 합산 (DB에는 세부 유지) |
| 10 | 부산 지명 충돌 예외 | 김해→김해공항, 동해→동해선/동해남부, 대구→해운대구 등은 부산 관련이므로 배제하지 않음 |
| 11 | **dcsnCntrctNo 차수 중복제거** | **확정계약번호 앞8자리(기관+공종) 그룹핑, 최신 차수(끝2자리)만 남김. 공사/용역/물품 공통 적용** |

### 3. 필터 적용 순서 (공사)
```
① 공사현장 필터 (bid_notices_raw 조인, 부산 외 현장 배제)
② 전화번호 + 키워드 필터 (타지역 의심 시 배제)
③ 복원 루트: 낙찰정보 브릿지 → 입찰공고 rgnLmtInfo (부산 제한이면 살림)
```

### 4. 필터 적용 순서 (용역)
```
⓪ 기관 그룹 판별: 부산시 소속기관 → 현장=부산 확정 (현장 필터 스킵)
                  정부/국가공공기관 → 아래 필터 적용
① 용역현장 필터 (조달요청 API cnstrtsiteRgnNm/rprsntDlvrPlce, 부산 외 배제)
② 전화번호 + 키워드 필터 (타지역 의심 시 배제)
③ 복원 루트: 낙찰정보 브릿지 → 입찰공고 rgnLmtInfo (부산 제한이면 살림)
```

### 4b. 필터 적용 순서 (물품)
```
① 전화번호 + 키워드 필터 (타지역 의심 시 배제)
② 복원 루트: 낙찰정보 브릿지 → 입찰공고 rgnLmtInfo (부산 제한이면 살림)
```

### 4c. 필터 적용 순서 (쇼핑몰)
```
① 기관 3그룹 분류 (부산전용 928개 / 혼재 64개 / 타지역전용 11개)
② 타지역 전용 기관의 공사자재(cnstwkMtrlDrctPurchsObjYn='Y') 배제
③ 혼재 기관: dlvrReqNm ↔ bidNtceNm 텍스트 매칭 → 타지역 현장 배제
```
※ 공사자재가 아닌 일반 쇼핑몰 품목은 필터 없음

### 5. 계약방식별 현장 파악 원칙 (2026-03-10 확정)

#### 경쟁입찰 (일반경쟁/제한경쟁/지명경쟁)
- 100% 입찰공고를 거침 → **공고 데이터에서 현장 파악 가능**
- 타지역 현장 확인 시 → **총 계약 대상(분모·분자)에서 제거**

#### 수의계약
- 입찰·공고 없음 → **현장 직접 확인 불가**
- 업체 정보(corpList)로 부산 업체 수주율은 파악 가능
- 단, 현장이 부산이라는 보장 없음 → **키워드+전화번호 필터로 보완**

#### 분야별 현장 데이터 소스
| 분야 | 데이터 소스 | 사유 |
|------|-----------|------|
| 공사 | `bid_notices_raw` (입찰공고 108만건) | 공고 API에 `cnstrtsiteRgnNm` 100% 제공 |
| 용역 | `servc_site.db` (조달요청 API) | 입찰공고에 현장 필드 없음, 조달요청에만 있음 |
| 물품 | 해당없음 | 현장 개념 없음 (키워드만) |
| 쇼핑몰 | **bid_notices_raw 텍스트 매칭** | 혼재 기관 dlvrReqNm↔bidNtceNm |

> ⚠️ **공사 수의계약은 조달요청 API에도 없음** — 98.3%가 조달청 미경유(기관 직접 처리)로 확인됨 (2026-03-10 테스트)

#### 4가지 분석 항목 모두 동일 원칙 적용
1. 지역업체 수주율 현황
2. 수주 상위/하위 수요기관
3. 분야별 주요 유출품목
4. 보호제도 미적용 (발주기관별, 공사/용역, 수의/경쟁)

#### 집에서 서버 배포 (paramiko SSH — 2026-03-28 확인)
- Windows SSH 클라이언트는 stdin 비밀번호 입력 불가 → **paramiko** 사용

import paramiko
client = paramiko.SSHClient()

print(stdout.read().decode('utf-8'))

### 왜 `core_calc.py` 공통 모듈을 만들었나?
- **문제**: `rate_calc_db.py`에 필터를 만들었는데, `build_api_cache.py`를 별도 작성하면서 필터 빠뜨림

- **해결**: 공통 모듈로 분리 → import만 하면 모든 필터 자동 적용
- **효과**: 필터전 42.1% → 필터후 66.5%

### 왜 시/군 단위 키워드를 추가했나?
- **문제**: 광역시/도 키워드만 있어서 포항/광양/안동/통영 등이 필터 통과
- **예시**: "포항영일만항 남방파제 축조공사" 956억이 부산 실적으로 잡힘
- **해결**: 83개 키워드 추가 + 부산 충돌 예외(김해공항, 동해선)
- **주의**: 김해, 양산은 부산 생활권 → BUSAN_EXCEPTIONS로 처리

### 왜 50억/30억/10억 차등 기준?
- **경과**: 처음 10억 → "시간 지나면 다 넘을것 같다" → 50억 제안
- **결과**: 공사 30억, 용역 30억, 물품/쇼핑몰 10억, 전체 50억
- **TOP_N**: 10위 → 사용자 요청으로 5위로 변경

### 왜 `compare_unit` 비교단위를 만들었나?
- **문제**: 같은 기관인데 코드가 여러 개 (예: 한국철도공사 부산지사/서부산역)
- **해결**: `compare_unit` 컬럼으로 통합 (자치구군→구군명, 산하기관→부모기관명)
- **교육청**: 대시보드에서 "부산교육청" 합산, DB에는 세부 유지

### 왜 전화번호+키워드 OR 조건?
- **초기**: AND 조건 → 전화번호만 다르고 계약명에 지명 없는 건이 빠져나감
- **변경**: OR 조건 (둘 중 하나만 충족해도 배제)
- **보완**: 낙찰정보 브릿지 + 입찰공고 rgnLmtInfo로 false negative 복원

### 왜 공사현장 소재지 필터?
- **문제**: 부산지방국토관리청 발주라 수요기관은 부산이지만 현장이 울릉/안동
- **해결**: `bid_notices_raw`의 `cnstrtsiteRgnNm`으로 부산외 현장 배제
- **한계**: 공사 59%가 ntceNo 없음(수의계약) → 키워드 필터로 보완

### 왜 낙찰정보 브릿지?
- **문제**: 전화번호/키워드 필터가 부산 실적까지 잘못 배제 (false negative)
- **해결**: `busan_award_*`에서 부산 지역제한 확정 공고번호 수집 → 복원

### 왜 `dminsttCd`(수요기관)로 매칭 변경했나?
- **문제**: `cntrctInsttCd`(계약기관) 기준이라 조달청 부산지방조달청이 매칭됨
- **결과**: 울산교육청/서울강서구 등 타지역 계약이 부산 실적으로 잡힘
- **해결**: `dminsttCd`(수요기관)로 1차 매칭 변경
- **효과**: 68.2% → 59.7%

### 왜 서울 구 이름을 키워드에 추가?
