## 1. 🏗️ 구축된 데이터 인프라 현황

현재 모든 기초 원본 데이터(Raw Data)는 로컬 SQLite 데이터베이스로 통합 관리되고 있습니다.

### ① 수요기관 마스터 DB (`busan_agencies_master.db`)
- **내용:** 전국 수요기관 중 부산광역시에 소재하거나 관련 있는 현행(Active) 기관 전체 목록.
- **용도:** 발주처가 부산 지역인지 판별하는 1차 필터 및 통계 그룹핑용.
- **데이터 건수:** 약 4,105건 (조달청 API `getDminsttInfo02` 기반)

### 2. SQLite 데이터베이스 (`busan_companies_master.db`) (신규 구축)
- **목적:** 기존 수동 엑셀 방식(`부산_지역업체_마스터.csv`)을 대체하는 부산 관내 조달업체 리스트 DB.
- **데이터 구조:** `company_master` 테이블 (사업자등록번호, 업체명, 지역코드, 개업일자, 나라장터 등록일자 등)
- **주요 특징:** 
  - 1984년~현재까지의 64.4만건 CSV 원본 데이터 중 부산+본사 업체 33,074건 1차 시드 적재
  - 1990년~현재 조달업체 API 연도별 전수조사 후 최신 정보 Overwrite (Upsert)
  - 최종적으로 부산+본사 기준 46,021개 조달업체 데이터를 O(1) 해시 룩업으로 실시간 지원

### 3. SQLite 데이터베이스 (`procurement_contracts.db`)
과거 데이터(2026.01.01 ~ 03.06) 전수 적재가 완료되었습니다.
- **총 적재 건수:** **814,642건** (전국 데이터)
- **세부 내역:**
  - 공사 계약 (`cnstwk_cntrct`): 82,856건
  - 용역 계약 (`servc_cntrct`): 207,348건
  - 물품 계약 (`thng_cntrct`): 49,738건
  - 종합쇼핑몰 납품요구 (`shopping_cntrct`): 474,700건

## 4. 🧠 고도화된 데이터 처리 기법

1. **지분율(%) 정밀 안분:** 공동수급 시 업체별 지분율을 `corpList`에서 파싱하여, 부산 업체가 실제 수주한 '파이(Pie)'만큼만 실적으로 인정하는 정밀 연산 적용.
2. **수요기관 마스터 DB 절대 기준 판별:** 수요기관의 주소 텍스트가 아닌, "수요기관코드", "사업자등록번호", "수요기관명"을 `수요기관 마스터 DB`와 직접 조인하여 부산 관내 발주처 여부를 100% 확정짓는 엄격한 기준 적용.
3. **관외 공사 자동 필터:** 부산 발주처 건이라도 공사 현장이 '진주, 대구' 등 타지역인 경우 텍스트 마이닝을 통해 통계 모수에서 자동 제외하여 수주율의 버블을 제거함.
